# Cisco Packet Tracer — Computer Networks Labs

Practical networking labs completed as part of the Computer Networks course 
at Sukkur IBA University, implemented and documented in Cisco Packet Tracer.

Each folder is a self-contained lab with its own topology, configuration 
steps, verification output, and relevant theory — documented as actually 
implemented, including noted gaps or limitations where applicable.

## Index

| # | Topic | Key Skills |
|---|-------|------------|
| [01](./01-ethernet-cabling) | Ethernet Cabling | TIA/EIA-568 standards, straight-through vs crossover |
| [02](./02-basic-connectivity-ip-config) | Basic Connectivity & IP Config | Static IP, ping diagnostics, hub vs switch behavior |
| [03](./03-network-devices-cli-basics) | Network Devices & CLI Basics | CLI config, hostname/MOTD, OSI Layer 1/2/3 devices |
| [04](./04-dhcp-network-services) | DHCP & Network Services | DHCP server config, FTP/TFTP/HTTP, client-server model |
| [05](./05-device-security-passwords) | Device Security | Console/enable/secret/VTY passwords, Telnet verification |
| [06](./06-spanning-tree-protocol) | Spanning Tree Protocol | Root bridge election, port states, STP/RSTP/PVRST/MST |
| [07](./07-static-routing-vlsm) | Static Routing & VLSM | Multi-router static routes, subnet exhaustion analysis |
| [08](./08-dynamic-routing-ospf) | Dynamic Routing (OSPF) | 4-router OSPF, multi-hop verification with TTL analysis |
| [09](./09-vlans-trunking-vtp) | VLANs, Trunking & VTP | VLAN segmentation, 802.1Q, VTP server/client sync |
| [10](./10-access-control-lists) | Access Control Lists | Standard ACLs, host-based filtering, control testing |
| [11](./11-nat-pat) | NAT & PAT | Dynamic NAT pool, PAT overload, translation table analysis |
| [12](./12-iot-network-configuration) | IoT Network Configuration | AAA/RADIUS, WPA2 Enterprise, IoT device registration |

## Tools Used
- Cisco Packet Tracer
- Cisco IOS CLI (Router/Switch)

## Notes
- `.pkt` source files are not renderable directly on GitHub; each folder's 
  README and screenshots document the configuration and verification in full.
- Where original lab submissions had gaps (missing values, incomplete steps, 
  or discrepancies between task instructions and actual configuration), these 
  are noted transparently within the relevant folder's README rather than 
  silently corrected.
